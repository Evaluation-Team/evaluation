A/B Testing

What is A/B testing?

A/B testing involves comparing two different versions of something to see which performs better.

It helps you understand how the differences between the two versions affect users’ behaviour and outcomes.

As such, A/B testing is useful for any digital health intervention that uses an app, website or newsletter campaign.

What can you A/B test?

You can A/B test almost anything that affects visitor behavior for example:

  • Headlines and text - including length, structure and position on the page
  • Content - including tone and language
  • Calls to action - including wording, size, colour and placement
  • Forms - including length, fields and descriptions
  • Images - like the choice between cartoon or realistic pictures

Pros

A/B tests let you explore different ideas and then make changes based on solid quantitative data. You will be able to know with statistical confidence what the impact of these changes will be for your users.

Cons

A/B tests can be complex to set up, and may need many users for the data to be statistically significant. But they are a powerful tool for making evidence-based design decisions.

Example: Couch to 5K

To do an A/B test with an app like Couch to 5K, you would design the test within the app itself. Typically, A is the existing design - called the ‘control’ - and B is the new design or variation on an element of the app.

For an app like Couch to 5K version A might be the existing instructions for using the app, and version B a new set of instructions the team wants to test. Users of Couch to 5K would be randomly allocated to either A or B, and this would decide the version of the instructions they would see.

After enough users have been through the experiment, it will be possible to tell with statistical confidence whether version A or B is performing better.


How to carry out A/B testing

1. Study your existing data. Using freely available analytic tools like Google Analytics, find problem areas in your intervention. For the running app Couch to 5K, for example, it appears users drop out of the programme in week 5.

2. Construct a hypothesis or a goal.Using all your insights and data, develop a hypothesis for your problem or construct a goal.

An example hypothesis could be:

“Adding push notifications encouraging users to run to week 5 of the Couch to 5K running programme will increase the number of people completing week 5 of the programme.” An example goal could be: “I want more users to go for a run after they receive a push notification.”

3. Test your hypothesis or goal. Create a new variation of your intervention that could help prove or disprove your hypothesis, then A/B test it against the original.

4. Decide how long to run the test. You should base this on the number of monthly visitors, the expected change in the user behaviour and other factors.

5. Create a control and variation. If you’re testing an existing web page or app, your control will be the original version. The new version will be your B.

6. Determine your sample size, if necessary. See an example sample size calculation tool.

7. Split your sample groups equally and randomly. If you can test with two or more groups that are relatively equal, your results will be more conclusive.

8. Analyse the test data. Analyse the results of your A/B test, comparing the outcomes of the original version against the variation. If there is an obviously better option from the A/B test, you can implement it. If the test results are inconclusive, you should review your hypothesis or goal, come up with new variations and continue A/B testing.

10. Report your results. Share what you’ve learned with your colleagues to inform their work.
